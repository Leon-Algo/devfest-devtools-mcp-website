GDG：Google Developer Groups

全球开发者在同一个坐标系里分享、实践与成长。

GDG 是什么？

GDG（Google Developer Groups） 是 Google 官方支持的全球性开发者社区。

这里聚集了学生、工程师、创业者和技术爱好者，大家通过线下 Meetup、Workshop 和 Hackathon 分享经验、探索新技术。

在这个控制室里，你可以把 GDG 想象成一张巨大的「节点网络」：世界各地的城市，是一个个发光的节点， DevFest 则是这些节点在同一季节的同步亮起。

GDG 网络的三个坐标

  • 🌍 全球覆盖：1000+ 城市，持续增长的本地开发者社区。
  • 👥 活跃成员：数百万开发者定期参加活动、提交代码、分享经验。
  • 🎓 技术分享：从入门到进阶，从产品到工程实践，主题持续更新。

DevFest：一年中亮度最高的那一段

每年一度的 DevFest，就像 GDG 网络的「高功率输出模式」。

聚焦 Google 技术

从 Android、Web、Cloud 到 AI，每一届 DevFest 都围绕最新的 Google 技术生态展开。

本地社区驱动

内容由本地 GDG 组织者和讲师策划，贴近所在城市开发者的真实需求。

实践优先

大量 Workshop 和 Labs，让参与者真正「动手敲键盘」，带走可落地的实践方案。

为什么是 DevTools MCP 这个 Workshop？

在众多选题里，我们选择了一条「AI + 浏览器 + 工程落地」的路径。

01 · AI + 浏览器自动化

借助 MCP 协议，AI 不再只是讲解 DevTools，而是可以真正接管浏览器： 打开页面、跑性能分析、检查错误、截图与体检。

02 · 标准化与开源生态

Chrome DevTools Protocol 与 Model Context Protocol 都是开放标准， 这意味着你在 Workshop 中写出的 Server，可以被多种 AI 平台复用。

03 · 实战路径清晰

这不是一场只讲概念的分享，而是一条从本地页面 → DevTools MCP → GitHub Actions → GitHub Pages 的实战链路。
